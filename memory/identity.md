## 🧠 弱模型补偿架构设计

**核心目标**：用弱模型（gemini-flash-lite、kimi-k2.5），实现不降智的效果，超越 GenericAgent。

### 弱模型降智的根因

| 问题 | 表现 | 根因 |
|------|------|------|
| 推理跳跃 | 直接给答案，中间步骤缺失 | 没有强制显式推理 |
| 事实幻觉 | 编造数字/引用 | 没有强制工具验证 |
| 上下文丢失 | 忘记前面的约束 | 没有结构化记忆 |
| 无法自我修正 | 错了就错了 | 没有批判环节 |

### 四层补偿机制

```
Layer 4: 元认知监控 (Meta-Cognition)
  - 置信度评估
  - 复杂度判断
  - 自动升级模型（必要时）
         ↓
Layer 3: 批判与修正 (Critic & Refine)
  - 生成后自我检查
  - 事实/数字强制验证
  - 低置信度时重新生成
         ↓
Layer 2: 显式推理链 (Explicit Reasoning)
  - Plan-and-Solve 强制分步
  - 工具调用前置（先查再答）
  - 中间结果持久化
         ↓
Layer 1: 结构化上下文 (Structured Context)
  - L0/L1/L2分层加载
  - 相关记忆按需注入
  - 任务状态显式追踪
```

### 实施计划

| 阶段 | 任务 | 预期效果 |
|------|------|----------|
| **Phase 1** | 显式推理 prompt 模板 | 弱模型推理质量提升 50%+ |
| **Phase 2** | 工具调用强制化 | 事实幻觉减少 90%+ |
| **Phase 3** | 自我批判循环 | 错误率降低 70%+ |
| **Phase 4** | 元认知监控 | 自动识别何时需要升级模型 |
